# 测试MySQL

## 概述

[SQL注入](https://owasp.org/www-community/attacks/SQL_Injection)漏洞发生在输入用于构造SQL查询而没有充分约束或清理时。使用动态SQL（通过字符串连接构造SQL查询）为这些漏洞打开了大门。SQL注入允许攻击者访问SQL服务器。它允许在用于连接数据库的用户权限下执行SQL代码。

*MySQL服务器*有一些特殊性，因此某些漏洞利用需要专门为此应用程序定制。这是本节的主题。

## 如何测试

当在使用MySQL数据库支持的应用程序中发现SQL注入漏洞时，根据MySQL版本和用户在DBMS上的特权，可以进行多种攻击。

MySQL至少有四个在生产中使用的版本：`3.23.x`、`4.0.x`、`4.1.x`和`5.0.x`。每个版本都有一组与版本号成比例的功能。

- 从4.0版开始：UNION
- 从4.1版开始：子查询
- 从5.0版开始：存储过程、存储函数和名为`INFORMATION_SCHEMA`的视图
- 从5.0.2版开始：触发器

应该注意的是，对于4.0.x之前的MySQL版本，只能使用布尔或基于时间的盲注入，因为未实现子查询功能或`UNION`语句。

从现在开始，我们假设存在一个经典的SQL注入漏洞，可以通过类似于[测试SQL注入](05-Testing_for_SQL_Injection.md)部分中描述的请求触发。

`https://www.example.com/page.php?id=2`

### 单引号问题

在利用MySQL功能之前，必须考虑在语句中如何表示字符串，因为Web应用程序经常转义单引号。

MySQL引号转义如下：

`'A string with \'quotes\''`

也就是说，MySQL将转义的单引号`\'`解释为字符而不是元字符。

因此，如果应用程序需要使用常量字符串才能正常工作，则需要区分两种情况：

1. Web应用程序转义单引号`'` => `\'`
2. Web应用程序不转义单引号`'` => `'`

在MySQL中，有一种标准方法可以绕过对单引号的需求，无需单引号即可声明常量字符串。

假设我们想知道字段`password`的值，其条件如下：

1. password like `'A%'`
2. ASCII值在连接的十六进制中：
   `password LIKE 0x4125`
3. char()函数：
   `password LIKE CHAR(65,37)`

### 多个混合查询

MySQL库连接器不支持用`;`分隔的多个查询，因此无法像Microsoft SQL Server那样在单个SQL注入漏洞中注入多个不同类的SQL命令。

例如，以下注入将导致错误：

`1 ; update tablename set code='javascript code' where 1 --`

### 信息收集

#### MySQL指纹识别

当然，第一件事是确定是否存在MySQL DBMS作为后端数据库。MySQL服务器有一个功能，用于让其他DBMS忽略MySQL方言中的子句。当注释块`'/**/'`包含感叹号`'/*! sql here*/'`时，它由MySQL解释，并被其他DBMS视为正常注释块，如[MySQL手册](https://dev.mysql.com/doc/refman/8.0/en/comments.html)中所解释。

示例：

`1 /*! and 1=0 */`

> 如果存在MySQL，则将解释注释块内的子句。

#### 版本

有三种方法可以获取此信息：

1. 使用全局变量`@@version`
2. 使用函数[VERSION()](https://dev.mysql.com/doc/refman/8.0/en/information-functions.html#function_version)
3. 使用带版本号的注释指纹识别`/*!40110 and 1=0*/`

意思是

```sql
if(version >= 4.1.10)
   add 'and 1=0' to the query.
```

这些是等效的，因为结果是相同的。

带内注入：

`1 AND 1=0 UNION SELECT @@version /*`

推理注入：

`1 AND @@version like '4.0%'`

响应将包含类似以下内容：

`5.0.22-log`

#### 登录用户

MySQL服务器依赖两类用户。

1. [USER()](https://dev.mysql.com/doc/refman/8.0/en/information-functions.html#function_user)：连接到MySQL服务器的用户。
2. [CURRENT_USER()](https://dev.mysql.com/doc/refman/8.0/en/information-functions.html#function_current-user)：执行查询的内部用户。

1和2之间有一些区别。主要区别是匿名用户可以（如果允许）使用任何名称连接，但MySQL内部用户是空名称（''）。另一个区别是，存储过程或存储函数作为创建者用户执行（如果在其他地方没有声明的话）。这可以通过使用`CURRENT_USER`知道。

带内注入：

`1 AND 1=0 UNION SELECT USER()`

推理注入：

`1 AND USER() like 'root%'`

响应将包含类似以下内容：

`user@hostname`

#### 当前使用的数据库

有本机函数`DATABASE()`

带内注入：

`1 AND 1=0 UNION SELECT DATABASE()`

推理注入：

`1 AND DATABASE() like 'db%'`

> 预期结果，类似以下字符串：
>
> `dbname`

#### INFORMATION_SCHEMA

从MySQL 5.0开始，创建了一个名为[INFORMATION_SCHEMA](https://dev.mysql.com/doc/refman/8.0/en/information-schema.html)的视图。它允许我们获取有关数据库、表和列以及过程和函数的所有信息。

| INFORMATION_SCHEMA中的表 | 描述 |
|------------------------------|-------------|
| SCHEMATA  | 用户拥有（至少）SELECT_priv的所有数据库 |
| SCHEMA_PRIVILEGES  | 用户对每个数据库的权限 |
| TABLES  | 用户拥有（至少）SELECT_priv的所有表 |
| TABLE_PRIVILEGES | 用户对每个表的权限 |
| COLUMNS | 用户拥有（至少）SELECT_priv的所有列 |
| COLUMN_PRIVILEGES | 用户对每个列的权限 |
| VIEWS | 用户拥有（至少）SELECT_priv的所有列 |
| ROUTINES | 过程和函数（需要EXECUTE_priv） |
| TRIGGERS | 触发器（需要INSERT_priv） |
| USER_PRIVILEGES | 连接用户的权限 |

所有这些信息都可以使用SQL注入部分描述的已知技术进行提取。

### 攻击向量

#### 写入文件

如果连接的用户具有`FILE`特权且单引号未被转义，则可以使用`into outfile`子句将查询结果导出到文件。

`Select * from table into outfile '/tmp/file'`

注意：没有办法绕过包围文件名的单引号。因此，如果对单引号进行了一些清理（如转义`\'`），将无法使用`into outfile`子句。

这种攻击可用作带外技术，以获取有关查询结果的信息，或写入可由Web服务器目录内的文件执行的文件。

示例：

`1 limit 1 into outfile '/var/www/root/test.jsp' FIELDS ENCLOSED BY '//'  LINES TERMINATED BY '\n<%jsp code here%>';`

> 结果以`rw-rw-rw`特权存储，由MySQL用户和组拥有。
>
> 其中`/var/www/root/test.jsp`将包含：
>
> `//field values//`
> `<%jsp code here%>`

#### 从文件读取

`load_file`是一个本机函数，如果文件系统权限允许，可以读取文件。如果连接的用户具有`FILE`特权，则可以使用它来获取文件内容。可以使用之前描述的技术绕过单引号转义清理。

`load_file('filename')`

> 使用标准技术可以导出整个文件。

### 标准SQL注入攻击

在标准SQL注入中，您可以直接在页面上将结果显示为正常输出或MySQL错误。通过使用已经提到的SQL注入攻击和已经描述的MySQL功能，根据测试人员面对的MySQL版本，深度直接SQL注入可能很容易完成。

一个好的攻击是通过强制函数/过程或服务器本身抛出错误来获取结果。可以在[MySQL手册](https://dev.mysql.com/doc/mysql-errors/8.0/en/server-error-reference.html)中找到MySQL抛出的错误列表，特别是本机函数。

### 带外SQL注入

带外注入可以使用[`into outfile`](#write-in-a-file)子句完成。

### 盲SQL注入

对于盲SQL注入，MySQL服务器原生提供了一组有用的函数。

- 字符串长度：
  - `LENGTH(str)`
- 从给定字符串中提取子字符串：
  - `SUBSTRING(string, offset, #chars_returned)`
- 基于时间的盲注入：
  - BENCHMARK和SLEEP `BENCHMARK(#ofcycles,action_to_be_performed)`
  当基于布尔值的盲注入没有产生任何结果时，基准函数可用于执行时序攻击。
  参见`SLEEP()`（MySQL > 5.0.x）作为基准的替代方案。

有关完整列表，请参阅[MySQL手册](https://dev.mysql.com/doc/refman/8.0/en/functions.html)
