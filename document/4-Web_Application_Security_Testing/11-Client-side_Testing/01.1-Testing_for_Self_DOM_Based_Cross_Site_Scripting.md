# 测试自基于DOM的跨站脚本

## 概述

自基于DOM的跨站脚本是一种特定攻击，需要事先了解基于DOM的跨站脚本和成功的社会工程。术语"self"在此是对以下事实的引用：用户需要将有效载荷注入输入字段，从而自己执行漏洞。该漏洞是特定的，因为网站的CSP（内容安全策略）可以阻止脚本执行。

本场景将在以下方式中使用术语"接收器"：在计算中，接收器、事件接收器或数据接收器是设计用于从另一个对象或函数接收输入或事件的类或函数。因此，为了找到可能的漏洞，我们首先需要识别要测试的应用程序的接收器。

## 如何测试

测试自基于DOM的跨站脚本的过程如下：

1. 查找允许用户输入的易受攻击的接收器。
2. 一旦识别出可能的接收器，可以插入有效载荷。
3. 检查浏览器开发者工具中的错误日志以查看结果，并得出结论。
4. 检查攻击者是否能够说服用户在不需要广泛技术知识的情况下插入有效载荷。

### 示例

这个特定示例来自此[hackerone工单](https://hackerone.com/reports/406587)。

在此示例中，以下JavaScript函数在网站`https://example.com`上执行。

```js
//Marketo Form Code
function strip(html) {
    var tmp = document.createElement("DIV");
    tmp.innerHTML = html;
    return tmp.textContent || tmp.innerText || "";
}

$('form').submit(function() {
    $('textarea').val(function() {
        return strip($(this).val());
    });
});
```

此功能的滥用可以描述如下：

1. `submit`事件处理程序将任何`textarea`元素的当前值传递给`strip`函数。
2. 此函数创建一个新的`div`元素并将其`innerHTML`属性设置为提供的值。
3. 最后，它返回结果`div`的`textContent`属性。

这种代码通常用于从字符串中删除HTML标签，因为`textContent`属性包含浏览器解析HTML时呈现的字符串。这种特定方法本质上是不安全的，因为它使用`innerHTML`。当向`innerHTML`属性提供用户输入时，它由Web浏览器解析，因此可能导致恶意JavaScript执行。

以下有效载荷可用于测试漏洞。`<img src=x onerror=alert(1) />`

开发者控制台将显示两个错误：一个指示请求了`https://www.example.com/x`并返回404（由于img标签的src属性）。另一个报告网站CSP的违规。

第二个错误发生是因为浏览器尝试执行`onerror`属性中的JavaScript代码，但网站的CSP阻止了执行。在禁用CSP的浏览器中执行相同操作允许执行`onerror`属性中的JavaScript。

攻击者可以通过说服用户将恶意有效载荷粘贴到联系表单的"消息"字段中，然后单击"发送消息"按钮来利用此漏洞。可以通过说服用户使用不支持CSP的浏览器版本来增强此攻击。
